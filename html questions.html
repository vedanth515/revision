Html interview questions !

1.What is Html!

HTML stands for Hypertext Markup Language, and it's the code that's used to structure and define the content of a web page.


2.What is the Html!

HTML stands for Hyper Text Markup Language. HTML is the standard markup language for creating Web pages. HTML describes the structure of a Web page. HTML consists of a series of elements. HTML elements tell the browser how to display the content.


3.What is Doctype in Html!

DOCTYPE, or document type declaration, is a line of code that tells a web browser which version of HTML a document is written in. It's the first line of every HTML or XHTML document.


4.What are Root elements of Html!

The root element of an HTML document is the Html tag, The Html tag is the top-level element of an HTML document, and all other elements are descendants of it.


5.What is Tag!

.HTML tags are the keywords on a web page that define how your web browser must format and display your web page.


6.Give some examples of self-closing tags!

Here are some examples of self-closing tags in HTML:
< br />: Inserts a line break
< hr />: Creates a horizontal rule break
< img />: Inserts an image < /p>
Self-closing tags, also known as empty tags, are HTML elements that are made up of a single tag and do not require a closing tag. They are used for shorter elements.


7.What is element!

The HTML element is everything from the start tag to the end tag: < tagname >Content goes here... Examples of some HTML elements: < h1 >My First Heading< /h1 >


8.How many types of elements are there in html!

Raw text elements: A type of HTML element
Void elements: A type of HTML element
Block-level elements: A type of HTML element
Inline elements: A type of HTML element that exists in line with its parent element


9.What is the difference between Inline and Blocl elements!

Block elements always start from a new line. Inline elements never start from a new line. Block elements cover space from left to right as far as it can go.

Inline elements only cover the space as bounded by the tags in the HTML element.


10.What is the difference between Semantic element and non-semantic element!

Semantic tags carry meaning and convey the structure and context of the content, making it easier for both developers and search engines to understand the purpose of each element.

Non-semantic tags, on the other hand, are used for formatting and styling purposes without conveying specific meaning.


11.What is href (vs) src attributes!

HREF stands for HyperText Reference, while SRC stands for Source Reference. HREF is used to refer to another HTML element.

whereas SRC refers to a non-HTML file. In other words, HREF is used to refer to pages on the same website, whereas SRC refers to pages on different servers.


12.What are the features of Html 5!

The new benefits of HTML5 and improved capabilities benefit users and developers, including faster page loading times, improved multimedia support, mobile optimization, and cross-browser compatibility. HTML5's cleaner code, improved semantics, and better user experience improve SEO and search rankings.


13.What are Entities in Html!

HTML entities are the reserved characters that have special meaning when used in an HTML document. Each of these codes starts with an ampersand and ends with a semicolon.


14.What is difference between Div (vs) Span!

The < span > tag is an inline container used to mark up a part of a text, or a part of a document. The < span > tag is easily styled by CSS or manipulated with JavaScript using the class or id attribute. The < span > tag is much like the < div > element, but < div > is a block-level element and < span > is an inline element


15.What are list tags in Html!

HTML lists come in three main categories: unordered lists, ordered lists, and definition lists. Each type serves a specific purpose and can be customized to fit your design and content needs.


16.What is attribute! and mention few attributes for img tags!

An attribute is use to give the extra information of the Tag. tag creates a holding space for the referenced image.

The < img > tag has two required attributes: src - Specifies the path to the image. alt - Specifies an alternate text for the image, if the image for some reason cannot be displayed.


17.What is thead in Table!

The < thead > tag is used to group header content in an HTML table. The < thead > element is used in conjunction with the < tbody > and < tfoot > elements to specify each part of a table (header, body, footer


18.What is th,td, and tr!

An HTML table consists of one < table > element and one or more < tr >, < th >, and < td > elements.

The < tr > element defines a table row, the < th > element defines a table header, and the < td > element defines a table cell.


19.What are meta tags!

.Meta tags in HTML are code tags that provide information about a web page to search engines and other clients. They are added to the < head > section of an HTML page and do not affect how the page is displayed.


20.Can we use th in tbody!

It's not valid HTML to place < th > elements directly inside < tbody >. The correct structure is to place < th > elements within < thead > and then < td > elements within < tbody >. This separation helps browsers and assistive technologies understand the structure of the table and improve accessibility.


21.What is Attribute of Html Tag!

An HTML attribute is a piece of markup language used to adjust the behavior or display of an HTML element. For example, attributes can be used to change the color, size, or functionality of HTML elements.


22.What is iframe and Anchor Tags!

An anchor is the visible region which must be selected to activate the link.

The iframe tag is used to display a web page inside a web page. When you create a document to be inside an iframe, any links in that frame will automatically open in that same frame.

35.Can we use block element in inline elements!

No, generally, you should not place block-level elements inside inline-level elements according to HTML standards. Doing so violates proper HTML structure and may cause unexpected behavior or rendering issues in browsers.

On the other hand, relative paths point to a file relative to the current page. Relative file paths start with ./ , followed by the path to the local file.
